## 1. IssuesとProjectsの違い・どちらを使うか

### 結論：両方セットで使う

IssuesとProjectsは**別物ではなく連携して動く**ものです。

```
Issue    = タスクの「中身」（詳細・コメント・担当者・議論の場）
Projects = タスクの「状態」を視覚化するボード（全体の進捗把握）
```

### 機能の違い

| 機能 | Issues | Projectsボード |
|---|---|---|
| 見た目 | リスト形式 | カンバン形式（視覚的） |
| 主な用途 | タスクの詳細・議論・報告 | 全体の進捗を一目で把握 |
| コメント・議論 | ○ | ✗（Issueに飛んで書く） |
| 担当者の設定 | ○ | ○（Issueと連動） |
| ステータス管理 | ラベルで管理 | 列をドラッグで管理 |
| 向いている場面 | 1件を深く確認・報告したい | 全タスクの状況を俯瞰したい |

### 使い分けのイメージ

```
タスクの登録       → Issue を作成
                         ↓（自動でボードにも追加される）
毎朝の進捗確認     → Projectsボードで全体を俯瞰
作業中の進捗報告   → Issue にコメントを投稿
作業完了           → ボードで「完了」列に移動 → Issue を Close
```

### どちらで操作するか迷ったら

| やりたいこと | 使う場所 |
|---|---|
| 新しいタスクを登録する | **Issues** で作成 |
| 全体の進捗を確認する | **Projectsボード** |
| 詳細・コメントを見る・書く | **Issues** |
| 状態（未開始→進行中）を変える | **Projectsボード**（ドラッグ） |
| タスクを完了にする | ボードで「完了」列へ移動 ＋ **Issue を Close** |

## 2. IssueとProjectsを連携させる方法（重要）

> ⚠️ GitHubのProjectsはデフォルトではIssueが自動表示されません。
> 以下のどちらかで設定が必要です。

### 方法A：Issueから手動で追加（1件ずつ）

1. Issueを開く
2. 右サイドバーの **[Projects]** をクリック
3. ボード名を選択



### 方法B：自動化を設定する（推奨・一度だけ設定）

一度設定すれば、以降はIssueを作成するたびに自動でボードに追加されます。

1. Projectsボードを開く
2. 右上 **[...]** → **[Workflows]** をクリック
3. **[Auto-add to project]** を選択
4. フィルター条件（例：`is:issue`）を設定 → **[Save and turn on workflow]**

**フォームの記入例（タスクの場合）**

| 項目 | 説明 | 例 |
|---|---|---|

| 完了条件 | チェックリスト形式で | `- [ ] ボタン色を統一` |

記入後 **[Submit new issue]** で作成完了。

## 4. Projectsボードとは

Issueを**カード**として視覚的に管理できるカンバンボードです。
Trello・Notionのボード機能と同じイメージです。

```
┌──────────┬──────────┬──────────┐
│  未開始  │  進行中  │   完了   │
├──────────┼──────────┼──────────┤
│ タスクC  │ タスクA  │ タスクX  │
│ タスクD  │ タスクB  │ タスクY  │
└──────────┴──────────┴──────────┘
```

| ステータス | 意味 |
|---|---|
| **未開始** | まだ着手していない |
| **進行中** | 現在作業中 |
| **完了** | 終わったタスク |

## 5. Projectsボードの作り方

> ※ Projectsボードは GitHub上で手動で1回だけ作成が必要です

1. リポジトリの **[Projects]** タブ → **[Link a project]** → **[New project]**
2. **[Board]** テンプレートを選択 → プロジェクト名を入力 → **[Create project]**
3. デフォルトの列を以下のように変更する：

   | 変更前 | 操作 | 変更後 |
   |---|---|---|
   | Todo | Rename | **未開始** |
   | In Progress | Rename | **進行中** |
   | Done | Rename | **完了** |
   | Backlog | Delete | （削除） |
   | Review | Delete | （削除） |

4. リポジトリと連携：プロジェクト設定 → **[Add repository]** でこのリポジトリを追加
5. 自動化の設定：[方法B](#方法b自動化を設定する推奨一度だけ設定) を参考に設定する

## 6. タスクの状態を変える

### Projectsボードで変える（推奨）

ボードを開き、カードを**ドラッグ＆ドロップ**で列間を移動するだけ。
<img width="1079" height="684" alt="5" src="https://github.com/user-attachments/assets/32423b4f-0a8e-4508-9afd-6ea3d9c533fa" />

### Issueのラベルで補足する

| 状況 | 付けるラベル |
|---|---|
| 作業中 | `status: in progress` |
| レビュー待ち | `status: review` |
| ブロック中 | `status: blocked` |

完了したら Issue 下部の **[Close issue]** をクリック。

## 8. メンバーを招待する / 権限を与える

**Settings → Collaborators → [Add people]**

ユーザー名またはメールアドレスで招待できます。
相手が承認するとアクセスできるようになります。

### 権限の種類

| ロール | できること |
|---|---|
| **Admin（管理者）** | 設定変更・メンバー管理・リポジトリ削除も可能 |
| **Maintain** | ほぼ全操作（設定・削除以外） |
| **Write** | Issue・PRの作成・編集・クローズ |
| **Read** | 閲覧のみ |

> 管理者権限を与えるには、招待後に相手の名前の横のロールを **Admin** に変更してください。

## 9. ラベル一覧

| ラベル | 意味 |
|---|---|
| `priority: high` | 優先度：高 |
| `priority: medium` | 優先度：中 |
| `priority: low` | 優先度：低 |
| `type: bug` | バグ修正 |
| `type: feature` | 新機能追加 |
| `type: task` | 一般タスク |
| `type: question` | 質問・相談 |
| `status: in progress` | 作業中 |
| `status: blocked` | ブロック中 |
| `status: review` | レビュー待ち |
